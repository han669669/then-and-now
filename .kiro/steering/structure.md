## File Naming Conventions
- **Components**: PascalCase (e.g., `ImageUploader.tsx`, `SettingsPanel.tsx`)
- **Hooks**: camelCase with `use` prefix (e.g., `useImageState.ts`)
- **Utilities**: camelCase (e.g., `canvas.ts`, `geometry.ts`)
- **Types**: camelCase (e.g., `image.ts`, `settings.ts`)
- **Tests**: Match source file name with `.test.ts` suffix

## Module Organization
- **Component-Based**: Each major feature is a self-contained component
- **Hook Extraction**: Reusable logic extracted into custom hooks
- **Utility Separation**: Pure functions isolated in `utils/`
- **Type Centralization**: Shared types in `types/` folder
- **Style Variants**: Centralized variant definitions using `tailwind-variants`

## Configuration Files
- **package.json**: Dependencies and scripts (Bun-compatible)
- **tsconfig.json**: TypeScript compiler configuration (strict mode)
- **vite.config.ts**: Vite build and dev server configuration
- **tailwind.config.js**: TailwindCSS theme and plugin configuration
- **.eslintrc**: ESLint rules for code quality
- **.prettierrc**: Prettier formatting rules

## Documentation Structure
- **README.md**: Project overview, setup, and usage instructions
- **DEVLOG.md**: Development timeline and decision log (for hackathon)
- **docs/**: Additional documentation (if needed)
  - API documentation
  - Component documentation
  - Architecture diagrams

## Asset Organization
- **public/**: Static assets served as-is (favicon, sample images)
- **src/assets/**: Assets imported by components (icons, default images)
- **Naming**: Descriptive kebab-case (e.g., `sample-before.jpg`, `arrow-icon.svg`)

## Build Artifacts
- **dist/**: Production build output (generated by Vite)
- **node_modules/**: Dependencies (managed by Bun)
- **.gitignore**: Excludes `dist/`, `node_modules/`, `.env`, etc.

## Environment-Specific Files
- **No environment variables needed**: Pure client-side application
- **Development**: `bun run dev` uses Vite dev server
- **Production**: `bun run build` generates optimized static files
- **Preview**: `bun run preview` serves production build locally
